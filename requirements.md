# 订单管理系统需求说明（重构版）

## 一、项目目标

本系统用于支撑风扇类产品的订单录入、订单组件展开、订单发货跟踪与物料库存判断，核心目标是：

- 通过结构化产品资料提升下单效率
- 用标准 BOM 替代手工备注式配置
- 支持订单级附加配置调整
- 按物料维度管理库存
- 对未发货订单给出“满足发货条件”提示

本系统是轻量级桌面业务系统，不是完整 ERP。

## 二、业务对象定义

### 1. 产品大类型
产品大类型表示业务上的产品系列，例如：

- 飞月
- 金牛
- 仿派
- 今慕光语

每个产品大类型：

- 可维护多个基础配置类型
- 每种基础配置类型有对应配置价格
- 每种基础配置类型有对应标准 BOM 清单

### 2. 具体型号
具体型号是用于下单选择的实际 SKU，例如：

- 飞月185
- 飞月186
- 飞月187

每个具体型号必须：

- 归属于一个产品大类型
- 绑定一个灯罩名称
- 绑定一个灯罩单价

### 3. 基础配置
基础配置是产品大类型下可选择的标准配置类型，例如：

- A 配置
- B 配置
- C 配置
- D 配置

每条基础配置至少包含：

- 所属产品大类型
- 配置类型标识
- 配置价格

### 4. 标准 BOM
标准 BOM 是基础配置对应的标准组件清单。

BOM 组件至少包含：

- 物料名称
- 规格或描述
- 材质
- 颜色
- 数量
- 金额

标准 BOM 属于主数据模板，只能在主数据维护中修改，不能直接在订单中改写。

### 5. 订单
订单是业务跟踪的核心记录。

每条订单至少包括：

- 下单日期
- 客户名称
- 产品大类型
- 具体型号
- 基础配置类型
- 灯罩名称
- 灯罩单价
- 配置价格
- 下单数量
- 订单组件快照
- 发货状态

订单保存时，系统必须将标准 BOM 展开为该订单的实际组件快照。

### 6. 附加配置
附加配置本质上是订单级对标准 BOM 的调整，不属于主数据模板。

附加配置支持：

- 增加组件
- 删除组件
- 修改组件数量
- 增减灯罩
- 添加标准 BOM 中原本不存在的组件

附加配置只对当前订单生效，不回写标准 BOM。

## 三、下单流程要求

### 1. 下单输入流程
用户下单时按以下顺序操作：

1. 选择具体型号
2. 系统自动确定所属产品大类型
3. 用户选择该产品类型支持的基础配置
4. 系统自动带出灯罩及灯罩单价
5. 系统自动带出配置价格
6. 系统自动生成基础 BOM 清单
7. 用户按订单需要做附加配置调整
8. 保存订单与订单组件快照

### 2. 订单组件生成规则
- 订单组件初始来源于标准 BOM
- 灯罩应作为订单实际组件参与保存
- 附加配置调整后，系统保存最终订单组件结果
- 历史订单不受后续主数据变更影响

## 四、库存管理要求

### 1. 库存维度
库存按物料或组件维度管理，不按型号维度管理。

### 2. 库存功能
系统至少支持：

- 库存数量录入
- 库存数量查询
- 按未发订单统计物料需求
- 对单条订单判断是否满足发货条件

### 3. 可发货判定
当某条未发货订单所需的全部未发物料数量都小于等于当前库存数量时，系统应将该订单标记为“可发货”。

该提示应至少出现在：

- 库存管理页
- 发货登记页

### 4. 当前阶段不包含
当前阶段不强制要求：

- 自动锁库
- 自动扣减库存
- 安全库存预警
- 采购建议
- 多仓库管理

## 五、系统模块要求

### 1. 主数据维护
系统需要提供最小可用的主数据维护能力，至少包括：

- 产品大类型维护
- 具体型号维护
- 基础配置维护
- BOM 维护

### 2. 订单管理
系统需要支持：

- 新订单录入
- 订单组件快照保存
- 订单查询
- 订单明细查看

### 3. 发货管理
系统继续保留追加式发货思路，至少支持：

- 发货登记
- 发货历史记录
- 订单未发数量跟踪

### 4. 库存管理
系统需要支持：

- 物料库存维护
- 库存查询
- 未发需求统计
- 可发货订单提示

## 六、关键业务原则

- 主数据与订单快照分离
- 标准 BOM 不能直接在订单中修改
- 订单附加配置只影响当前订单
- 库存判断按物料维度进行
- 当前优先实现业务闭环，不提前实现复杂 ERP 特性

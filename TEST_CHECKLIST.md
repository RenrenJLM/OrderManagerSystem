# OrderManagerSystem 测试清单

用于手动回归当前正式版全部主功能。按顺序执行即可，不需要一次全做完。

## 1. 启动与数据库

- [ ] 启动程序后，默认进入 `订单录入` 页。
- [ ] 如果上次使用的数据库存在且结构正确，程序能自动打开该数据库。
- [ ] `文件 -> 新建数据库...` 可以创建新库，并正常切换到新库。
- [ ] `文件 -> 打开数据库...` 可以打开已有正式版数据库。
- [ ] `文件 -> 关闭当前数据库` 后，录单、发货、导入、导出、打印、主数据保存等操作被禁用。
- [ ] 在未打开数据库状态下，界面状态显示正常，不出现异常报错。

## 2. 产品资料维护

- [ ] 可以新增、修改、查看 `产品类型`。
- [ ] 可以新增、修改、查看 `具体型号`。
- [ ] 具体型号可以正确关联到所属产品类型。
- [ ] 可以新增、修改、查看 `基础配置`。
- [ ] 可以新增、修改、查看 `基础配置 BOM`。
- [ ] BOM 中的数量、单位、金额保存后再次打开仍正确显示。

## 3. 订单录入

- [ ] 在 `订单录入` 页选择具体型号后，产品类型会自动带出。
- [ ] 选择基础配置后，系统会自动生成订单组件。
- [ ] 灯罩名称和灯罩单价能自动带出。
- [ ] 修改订单数量后，订单组件需求数量会同步更新。
- [ ] 可以新增手工组件，并成功保存到订单。
- [ ] 将组件从库存候选切回手工输入时，不会残留旧的规格、材质、颜色、单位和单价。
- [ ] 保存订单后，在 `订单查询` 中可以查到该订单。
- [ ] 保存订单后，订单中的配置价格与灯罩价格正确。

## 4. 发货管理

- [ ] `发货管理` 页可以加载未完成订单。
- [ ] 可发货订单会显示 `可发货`，缺料订单会显示 `不可发货`。
- [ ] 执行整单发货后，订单状态、已发数量、未发数量会同步更新。
- [ ] 执行组件级发货后，只更新对应组件的已发数量和未发数量。
- [ ] 发货后，发货记录能在订单详情中看到。
- [ ] 发货后，订单查询页中的状态会同步变化。

## 5. 库存管理

- [ ] 可以新增、修改库存物料。
- [ ] 保存库存后，库存列表会立即刷新。
- [ ] 相同物料名称但不同 `unit_name` 时，可以分别保存为不同库存项。
- [ ] 发货后，对应库存数量会自动扣减。
- [ ] 同名不同单位的库存不会被错误扣减。
- [ ] 库存需求汇总能显示需求数量、库存数量和缺口数量。

## 6. 数据导入

- [ ] 可以成功导入 `product_categories` CSV。
- [ ] 可以成功导入 `product_skus` CSV。
- [ ] 可以成功导入 `base_configurations` CSV。
- [ ] 可以成功导入 `base_configuration_components` CSV。
- [ ] 可以成功导入 `inventory_items` CSV。
- [ ] 导入完成后，产品资料、库存、发货提示、订单查询相关页面会同步刷新。

## 7. 订单查询、导出与打印

- [ ] `订单查询` 支持按日期、客户、具体型号筛选。
- [ ] 选中订单后，可以看到订单组件明细。
- [ ] 选中订单后，可以看到发货记录。
- [ ] 可以导出 `订单汇总 CSV`。
- [ ] 可以导出当前订单的 `发货清单 CSV`。
- [ ] 可以导出 `库存需求汇总 CSV`。
- [ ] 可以打印当前订单。

## 8. 重点回归场景

- [ ] 新建数据库后，完整走通一次：维护主数据 -> 录入订单 -> 发货 -> 查询。
- [ ] 录入订单后，修改库存，再查看可发货状态，结果正确。
- [ ] 先做组件级发货，再做整单发货，状态和数量累计正确。
- [ ] 订单保存、发货、库存修改后，相关页面都能及时看到最新结果。

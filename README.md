# OrderManagerSystem

订单管理系统，用于管理产品资料、订单录入、发货登记、库存维护，以及订单查询、导入、导出和打印。

当前系统只使用现行新业务模型，不兼容旧模板数据。

## 核心功能

- 产品资料维护：
  - 产品类型
  - 具体型号
  - 基础配置
  - 基础配置 BOM
- 订单录入：
  - 选择具体型号与基础配置
  - 自动生成订单组件
  - 支持订单级附加组件调整
  - 保存订单组件快照
- 发货管理：
  - 整单发货
  - 组件级发货
  - 发货后同步更新订单状态、已发未发数量与库存数量
- 库存管理：
  - 维护库存资料
  - 汇总未发订单物料需求
  - 判断订单是否满足发货条件
- 查询、导出与打印：
  - 订单查询
  - 发货清单导出
  - 库存需求汇总导出
  - 当前订单打印

## 数据库文件管理

系统当前采用单一当前数据库连接。

### 启动规则

- 程序会尝试自动打开上次成功使用的数据库。
- 如果上次数据库不存在或结构不合法，程序会进入“当前未打开数据库”安全状态。

### 新建数据库

- 入口：`文件 -> 新建数据库...`
- 新建时只创建当前正式版所需的新模型表结构。
- 新建后会自动切换到该数据库。
- 新建数据库不会自动写入默认产品资料、BOM 或库存。

### 打开数据库

- 入口：`文件 -> 打开数据库...`
- 只允许打开符合当前正式版结构要求的 SQLite 数据库。
- 若缺少关键表或关键字段，系统会拒绝打开并提示正式中文错误。

### 关闭数据库

- 入口：`文件 -> 关闭当前数据库`
- 关闭后界面会进入安全状态。
- 在未打开数据库状态下，录单、发货、导入、导出、打印和主数据保存等依赖数据库的操作都会被禁用。

## 怎么使用

### 1. 先维护基础资料
建议先按下面顺序准备数据：

1. 产品类型
2. 具体型号
3. 基础配置
4. 基础配置 BOM
5. 库存资料

如果基础资料不完整，订单录入、发货和库存判断都会受影响。

### 2. 录入订单
在 `订单录入` 页：

1. 选择具体型号
2. 选择基础配置
3. 输入客户、日期和数量
4. 检查系统自动生成的订单组件
5. 按需要调整附加组件
6. 保存订单

保存后，系统会把当时的订单组件保存为订单快照，后续主数据变化不会影响历史订单。
订单主表中的配置价格与灯罩价格会按最终订单组件快照固化。

### 3. 发货
在 `发货管理` 页：

- 可做订单级发货
- 可做组件级发货
- 发货后会同步更新：
  - 订单状态
  - 组件已发数量 / 未发数量
  - 库存数量

### 4. 查询
在 `订单查询` 页可以：

- 按日期、客户、具体型号筛选订单
- 查看订单组件明细
- 查看发货记录
- 导出订单汇总
- 导出当前订单发货清单
- 打印当前订单

## 数据导入

系统支持 CSV 导入，入口在产品资料维护页和库存管理页。

### 导入顺序

建议按下面顺序导入：

1. `product_categories`
2. `product_skus`
3. `base_configurations`
4. `base_configuration_components`
5. `inventory_items`

### 各类导入文件说明

#### 1. 产品类型导入
对应业务数据：产品类型。

必填列：
- `name`

可选列：
- `is_active`

#### 2. 具体型号导入
对应业务数据：每个产品类型下可下单的具体型号，以及默认灯罩信息。

必填列：
- `product_category_name`
- `sku_name`
- `lampshade_name`
- `lampshade_unit_price`

可选列：
- `is_active`

#### 3. 基础配置导入
对应业务数据：每个产品类型下可选的基础配置及配置价格。

必填列：
- `product_category_name`
- `config_code`
- `config_name`
- `config_price`

可选列：
- `sort_order`
- `is_active`

#### 4. 基础配置 BOM 导入
对应业务数据：某个基础配置对应的组件清单。

必填列：
- `product_category_name`
- `config_code`
- `component_name`
- `quantity`
- `unit_amount`

常用列：
- `component_spec`
- `material`
- `color`
- `unit_name`
- `sort_order`

说明：
- 同一产品类型 + 配置代码下的 BOM 按整组覆盖更新。

#### 5. 库存导入
对应业务数据：库存物料、单价和库存数量。

必填列：
- `product_category_name`
- `component_name`
- `unit_price`
- `current_quantity`

常用列：
- `component_spec`
- `material`
- `color`
- `unit_name`
- `note`

说明：
- 同名物料在不同 `unit_name` 下会按不同库存项分别维护。

## 数据导出

### 1. 订单汇总导出
入口：`订单查询`

导出内容包括：
- 订单编号
- 日期
- 客户
- 产品类型
- 具体型号
- 基础配置
- 数量
- 发货状态
- 可发货状态

### 2. 发货清单导出
入口：`订单查询`

先选中一条订单，再导出当前订单发货清单。

导出内容包括：
- 订单基本信息
- 订单组件
- 每套数量
- 需求数量
- 已发数量
- 未发数量

### 3. 库存需求汇总导出
入口：`库存管理`

导出内容包括：
- 产品类型
- 物料名称
- 规格
- 材质
- 颜色
- 单位
- 需求数量
- 库存数量
- 缺口数量

## 主要数据表含义

### `product_categories`
产品类型主数据。

### `product_skus`
具体型号主数据。
每条记录对应一个可下单型号，并保存默认灯罩和灯罩单价。

### `base_configurations`
基础配置主数据。
定义某个产品类型下有哪些基础配置，以及对应配置价格。

### `base_configuration_components`
基础配置 BOM。
定义某个基础配置包含哪些组件、数量和金额。

### `orders`
订单主表。
保存客户、日期、产品类型、具体型号、基础配置、灯罩信息、配置价格和订单状态。

### `order_components`
订单组件快照表。
保存订单在下单当时的实际组件清单，以及已发数量、未发数量。

### `inventory_items`
库存表。
保存物料、单价、库存数量和备注。

### `structured_shipment_records`
发货记录表。
保存订单级发货和组件级发货历史。

## 使用建议

- 新系统首次使用时，先导入或维护完整基础资料，再开始录单。
- 导入 BOM 和库存前，先确认产品类型和具体型号已经存在。
- 导出发货清单前，先在订单查询中选中目标订单。

# OrderManagerSystem

`OrderManagerSystem` 是一个基于 **C++ + Qt 6 + Qt Widgets + CMake + SQLite** 的轻量级桌面订单管理系统，当前已完成订单录入、发货跟踪和最小查询闭环。

开发阶段记录见：`development_log.md`。  
开发过程中可使用 WSL 作为辅助分析环境，实际 Qt 编译、运行和功能验证统一以 Windows `Qt 6.7.3 + MSVC 2019` 为准。

---

## 当前已实现功能

### 1. 订单录入
- 支持录入订单日期、客户名称、产品型号、订单套数。
- 支持模板配置与自定义配置两种录入方式。
- 模板配置会自动展开模板组件并保存为订单组件快照。
- 自定义配置已改为基于当前产品型号的可选组件下拉选择：
  - 用户选择组件
  - 输入每套数量
  - 单价自动带出
  - 总需求数量与总价自动计算

### 2. 发货跟踪
- 支持订单级发货登记。
- 支持组件级发货登记。
- 发货采用追加记录模式，保留完整历史。
- 同步维护订单与订单组件的已发/未发数量。
- 支持机体独立追踪。

### 3. 订单查询
- 当前采用 3 个页签组织界面：
  - `订单录入`
  - `发货登记`
  - `订单查询`
- 订单查询页签支持最小查询闭环：
  - 查看订单列表
  - 查看订单组件明细
  - 查看发货记录
  - 在订单列表中查看单套总价与订单总价
  - 价格显示基于订单保存时的实际快照价格，不强制回退为模板默认价
  - 基础筛选仅包含：
    - 客户名称模糊
    - 产品型号
    - 仅未完成

## 当前数据库结构

当前项目已使用并验证的核心表包括：

- `product_models`
- `option_templates`
- `option_template_components`
- `product_model_components`
- `order_items`
- `order_item_components`
- `shipment_records`

其中：
- `option_templates` / `option_template_components` 用于模板配置
- `product_model_components` 用于自定义配置下拉组件来源
- `order_item_components` 保存订单组件快照
- `shipment_records` 保存订单级与组件级发货历史

---

## 当前系统特点

- 订单项独立管理，不自动合并原始订单数据
- 模板与实际订单组件快照分离
- 支持模板配置与自定义配置并存
- 自定义配置不再依赖手输组件名称和单价
- 支持订单级和组件级双层发货跟踪
- 查询功能保持最小范围，便于后续逐步扩展
- 保留历史记录，保证业务数据可追溯

## 项目定位

本项目不是完整 ERP 系统，而是围绕以下核心流程构建的轻量桌面管理工具：

- 订单录入
- 配置结构化管理
- 发货过程跟踪
- 最小订单查询

# OrderManagerSystem

`OrderManagerSystem` 是一个轻量级订单管理系统，面向日常业务中的订单录入、配置管理、发货跟踪、订单查询与汇总输出等场景。

开发阶段记录见：[development_log.md](development_log.md:1)
开发过程中使用 WSL 作为 Codex 辅助环境，实际 Qt 编译、运行与功能验证统一在 Windows `Qt 6.7.3 + MSVC 2019` 环境中完成。

---

## 系统用途

本系统主要用于解决以下问题：

- 订单录入效率低
- 产品配置依赖人工填写，容易出错
- 发货记录不清晰，未发数量不易跟踪
- 同一订单下的组件明细缺乏结构化管理
- 订单查询、统计与汇总输出不方便

系统的目标是将订单、配置和发货过程进行结构化管理，提高录单效率，增强数据可追溯性，并为后续业务扩展预留空间。

---

## 系统构成

本系统主要由以下几部分组成：

### 1. 订单管理
负责订单录入与订单项维护，支持按订单项进行独立跟踪。

### 2. 配置模板管理
根据产品型号提供常用配置模板，并支持维护模板对应的组件明细。

### 3. 自定义组件管理
支持在模板基础上对当前订单追加、修改或删除组件，形成订单实际组件数据。

### 4. 发货跟踪
支持订单级发货记录与组件级发货记录，实时统计已发数量和未发数量。

### 5. 查询与汇总
支持按客户、产品型号、日期范围、未发状态等条件进行查询，并生成订单汇总结果用于查看或输出。

---

## 核心特点

- 订单项独立管理，不自动合并原始数据
- 配置模板与实际订单组件分离
- 支持模板配置与自定义配置并存
- 支持订单级和组件级双层发货跟踪
- 保留历史记录，保证业务数据可追溯
- 面向实际业务使用，强调简单、清晰、可维护

---

## 项目定位

本项目是一个轻量级业务系统，重点关注：

- 订单录入效率
- 配置结构化管理
- 发货过程跟踪
- 订单查询与汇总

本项目不是完整 ERP 系统，而是围绕订单与发货核心流程进行实现的桌面端管理工具。
